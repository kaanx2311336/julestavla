`JulesCliService.cs`, lokal Jules CLI komutlarini calistiran servistir.

- `VersionAsync`, `jules.cmd version` ile CLI'nin hazir olup olmadigini kontrol eder.
- `CreateSessionAsync`, `jules.cmd new --repo <repo> "<prompt>"` ile Jules remote session baslatir.
- `ListSessionsAsync`, `jules.cmd remote list --session` ile tum Jules session durumlarini okur.
- `PullSessionAsync`, `jules.cmd remote pull --session <id>` ile tamamlanan session sonucunu ceker; `apply` true verilirse patch'i uygular.
- `ReplyToSessionAsync`, Jules CLI'nin Windows Credential Manager'daki `jules-cli:default` OAuth tokenini okuyup AIDA `tasks/{id}:interact` endpointine feedback gonderir.
- Reply akisi once Jules API'de calisan `feedbackGiven.feedback` payload formatini dener, sonra geriye uyumluluk icin `freeFormText` dener; basarili mod `CommandResult` ile raporlanir.
- Bu sayede `Awaiting User Feedback` ve `Awaiting Plan Approval` durumlari web ekranina tiklamadan ayni Jules session icinde cevaplanabilir.
- Komutlar proje klasoru calisma diziniyle calisir.
- Standart cikti, hata cikisi ve exit code `CommandResult` olarak UI'a tasinir.
- 2026-04-26 duzeltmesiyle Jules CLI exit code 0 verse bile `Failed to apply patch`, `error: patch failed` veya `error: exit status` gibi bilinen patch hata metinleri gorulurse `CommandResult.ExitCode` 1'e normalize edilir. Boylece ajan patch uygulanmadigi halde `AppliedThisTurn=true` yazmaz.
